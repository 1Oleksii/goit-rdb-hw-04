# Домашнє завдання 4 (goit-rdb-hw-04) 
**Тема:** DDL та DML команди, складні SQL запити

Цей репозиторій містить виконання практичного завдання з проєктування баз даних та аналітичних SQL-запитів.

## Основні етапи роботи

### 1. Робота з базою даних LibraryManagement
- Проєктування структури: створено схему бібліотеки з 5 таблиць: authors, genres, books, users, borrowed_books.
- Цілісність даних: налаштовано FOREIGN KEY, використано AUTO_INCREMENT, NOT NULL, типи INT, VARCHAR, YEAR, DATE.
- Заповнення даними: таблиці наповнені тестовими записами (включаючи NULL для книг без автора, наприклад «1984»).

### 2. Складні запити до бази goit_hw3
- Об’єднання таблиць: написано запит з INNER JOIN 8 таблиць (order_details, orders, customers, products, categories, employees, shippers, suppliers).
- Порівняння JOIN: INNER, LEFT, RIGHT — кількість рядків однакова (518), бо в базі повна посилальна цілісність, немає NULL у зовнішніх ключах.
- Аналітика та групування:
  - Відфільтровано замовлення працівників з employee_id від 4 до 10.
  - Згруповано за категоріями, пораховано кількість рядків і середню кількість товару.
  - Застосовано HAVING (середня кількість > 21), сортування за спаданням, вибірку з пропуском першого рядка (LIMIT 4 OFFSET 1).

## Технологічний стек
MySQL Workbench, SQL (DDL, DML, Aggregation functions, JOIN, GROUP BY, HAVING, LIMIT/OFFSET)
